1. The late-fan problem
Football fans often arrive late, lose connectivity, or want context without having the result spoiled. PulseProof is the trusted second screen that lets them understand only what has happened so far, keep the recap, and preserve a source-bound memory.

2. Real deployed product walkthrough
This is the deployed product, not a mock-up. The selected World Cup replay is clearly labelled as a published-report replay. The walkthrough enables Spoiler Shield, then presses the same Start Catch-up control available to every fan. The timeline advances from the visible event prefix, so future goals, cards and VAR cannot leak. Share Safe Relay then commits that exact prefix and returns an Ed25519-signed Catch-up Capsule with zero future-event payloads. Next, PulseProof stores a sanitized offline recap. Finally, the browser requests a new short-lived attestation and verifies its signature locally.

3. Eight production checks
Now the Judge Room tests production itself. It confirms TxLINE devnet credentials, separates live fixtures from labelled replays, opens the public score stream without polling, and proves a two-event prefix cannot see the future. It then issues, redeems and verifies a signed fan relay containing exactly those two events. The browser also verifies a fresh receipt, reads both executable Solana programs plus the finalized transaction, and confirms offline caching never captures APIs, SSE or proofs.

4. Public devnet evidence
The chain evidence is public and independently inspectable. This finalized devnet receipt was accepted by the deployed PulseProof program. The contract binds the wallet, fixture, source moment, evidence digest, points, badge and expiry, then creates a deterministic receipt PDA so the same memory cannot be replayed.

5. A defensible fan memory loop
Score apps, fast recaps and on-chain fandom already exist separately. PulseProof combines strict source lanes, a signed no-spoiler fan relay, prefix-safe Catch-up, a sanitized offline pack, non-transferable anti-replay memories, and a public SSE-to-Solana flight recorder. The app, source code, live tests, captions, transaction and technical documentation are all available from the Judge Room.
