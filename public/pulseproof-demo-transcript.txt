1. The late-fan problem
Football fans often arrive late, lose connectivity, or want context without having the result spoiled. PulseProof is the trusted second screen that lets them understand only what has happened so far, keep the recap, and preserve a source-bound memory.

2. Real deployed product walkthrough
This is the deployed product, not a mock-up. The selected World Cup replay is clearly labelled as a published-report replay. The walkthrough first enables Spoiler Shield, so finished scores disappear everywhere. It then presses the same Start Catch-up control available to a fan. The timeline advances from the visible event prefix, which means future goals, cards and VAR cannot leak into an earlier summary. Next, PulseProof stores a consumer-safe offline recap on this device. Raw feed actions and technical metadata are excluded. Finally, the browser requests a new short-lived attestation and verifies its Ed25519 signature locally. The visible evidence digest and expiry come from that fresh response.

3. Seven production checks
Now the Judge Room tests the production system itself. It confirms active TxLINE devnet credentials, separates live fixtures from labelled replays, opens the public score stream and receives ready plus pulse without polling. It proves a two-event Catch-up prefix cannot see future event IDs, verifies a fresh Ed25519 receipt in this browser, reads both executable Solana programs and the finalized reference transaction, and confirms that offline caching never captures APIs, SSE or proofs.

4. Public devnet evidence
The chain evidence is public and independently inspectable. This finalized devnet receipt was accepted by the deployed PulseProof program. The contract binds the wallet, fixture, source moment, evidence digest, points, badge and expiry, then creates a deterministic receipt PDA so the same memory cannot be replayed.

5. A defensible fan memory loop
Score apps, fast recaps and on-chain fandom already exist separately. PulseProof combines strict source lanes, prefix-safe Catch-up, a sanitized offline pack, non-transferable anti-replay memories, and a public SSE-to-Solana flight recorder. The app, source code, live tests, captions, transaction and technical documentation are all available from the Judge Room.
